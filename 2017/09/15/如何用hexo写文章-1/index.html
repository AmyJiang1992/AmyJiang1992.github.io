如何用hexo写文章(1)

Veröffentlicht am 2017-09-15

文章目标人群：刚刚利用各路教程终于搭建好了Hexo，准备写下第一篇文章的同学们。
我的基本情况：hexo + Github pages搭建的博客，nexT主题，Mac OS Sierra操纵系统。

现在，让我们开始吧！

Step 0: 基本了解 + 准备工作

  1. 我们打开hexo所在目录 -> source文件夹 -> _posts, 可以看到一个hello-world.md文件。
  2. 回到hexo文件夹，就是里面有_config.yml, source, themes等等的那个。
  3. 用hexo server命令来进行本地预览，在浏览器输入http://localhost:4000/即可。

什么是md文件？这个_posts文件夹是一个比较特殊的文件夹，因为它装着所有你发布出去的文章。

Step 1: 新建文章

在hexo所在目录下，打开terminal，在命令行输入：

hexo new a

也可以给标题加上双引号,”a”。
通过这行命令，我们新建出来了一个page，而且是一个post page，page还有其他种，稍后我们会提到。
正确结果：我们会在_posts里看见多了一个a.md文件。因此我们也知道了，默认情况下，hexo为我们创建markdown文件。刷新页面（http://localhost:4000/）我们能看见新添一个名字叫a的文章，没有任何内容。

接下来我们打开a.md文件

title: 1
date: 2017-09-15 19:00:41

tags:

在这里随便写点什么

然后再刷新,就会看到你写的内容。与此同时，hexo也会自动为这个post生成一个页面，当我们点击标题，就会进入那个页面。

Step 2: 草稿箱

除了post page，我们还可以新建draft page，也就是草稿。很多时候我们需要先写成草稿暂不发布。draft page就可以满足我们的要求，我们的网站上是看不到草稿文件的。

输入 hexo new draft b
我们会在source下看见一个新的文件夹，_drafts，这个里面会装所有的草稿文件。

那如何才可以在不发布的情况下，预览一下文章在网站上显示的样子呢？

输入 hexo server –draft
当然，你要先shut down原来开着那个server，才可以开启新的server。如此一来，我们就可以预览草稿文件啦

Step 3: 发布草稿

当你准备好了要发布你的草稿时
输入 hexo publish b 就可以了
你会发现_drafts里的b.md不见了，跑到了_posts里面。

Step 4: normal page

我目前还不知道该如何用中文称呼这类页面。我们可以把post和draft统称为blog pages，在这之外的一种就是normal pages， 类似一个网站上的“关于”页面。
输入 hexo new page c 即可
和前两种不同，这个命令会在source文件夹内创建出c文件夹，与_posts，_drafts并列。里面有一个index.md文件。
刷新我们的个人网页，你会发现c并没有出现在页面内，那它在哪儿呢？你需要在网址加上c/， 即http://localhost:4000/c/
正因为c不是一个blog page，所以它也不会出现在blog列表中，要通过URL来显示。

Step 5: 小tip

现在我们了解到page一共有三种，post，draft，normal，那么大家可以想一下，问什么一开始的时候我们用hexo new a 就直接生成了post呢？
那是因为我们默认这样。
打开熟悉的_config.yml文件，找到default_layout: post。如果你习惯默认成草稿，那就把它改成draft就好。
